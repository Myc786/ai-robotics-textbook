## Code of Conduct

This project and everyone participating in it is governed by our Code of Conduct. By participating, you are expected to uphold this code.

## How Can I Contribute?

### Reporting Bugs
- Use the bug report template when creating an issue
- Check if the issue already exists before creating a new one
- Provide detailed steps to reproduce the issue

### Suggesting Enhancements
- Use the feature request template when suggesting enhancements
- Explain the enhancement and its use cases
- Consider if the enhancement fits the project scope

### Adding Textbook Content
- Follow the existing chapter structure
- Include code examples where appropriate
- Add exercises to reinforce learning

### Improving the Code
- Fix bugs
- Add new features to the RAG chatbot
- Improve documentation
- Enhance the user interface

## Development Setup

1. Fork the repository
2. Clone your fork: `git clone https://github.com/Myc786/ai-robotics-textbook.git`
3. Navigate to the project directory: `cd ai-robotics-textbook`
4. Install dependencies: `npm install`
5. For RAG backend: `cd RAG-backend && pip install -r requirements.txt`

## Textbook Content Contributions

When adding or modifying textbook content:

- Follow the existing markdown structure
- Include code snippets in appropriate language blocks
- Add exercises at the end of chapters
- Ensure content is educational and accurate
- Use consistent terminology throughout

## Code Contributions

### Frontend (Docusaurus/React)
- Follow React best practices
- Maintain consistency with existing code style
- Test components in multiple browsers

### Backend (RAG System)
- Follow Python PEP 8 guidelines
- Write clear, well-documented code
- Include error handling where appropriate

## Style Guides

### Markdown Style
- Use proper heading hierarchy (#, ##, ###)
- Format code blocks with appropriate language identifiers
- Use relative links for internal references

### Git Commit Messages
- Use present tense ("Add feature" not "Added feature")
- Use imperative mood ("Move cursor to..." not "Moves cursor to...")
- Limit first line to 72 characters or less
- Reference issues and pull requests after the first line

## Pull Request Process

1. Ensure your changes are well-tested
2. Update documentation as needed
3. Update the README if necessary
4. Fill out the pull request template
5. Submit the pull request against the `main` branch
6. Address any feedback from reviewers

## Questions?

If you have any questions, feel free to open an issue or contact one of the maintainers.

Thank you again for your contribution!
